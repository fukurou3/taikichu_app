# 『待機中。』アプリの現在の実装状況

## 📱 基本機能
- **カウントダウン作成・表示**: イベントまでの時間を表示
- **コメント機能**: Twitter風UI でのコメント投稿・表示
- **いいね機能**: カウントダウンへのいいね操作
- **トレンドランキング**: 人気カウントダウンの表示

## 🏗 データベース構造（Firestore）

### countsコレクション（カウントダウンの基本情報）
```
{
  eventName: string,           // イベント名
  eventDate: timestamp,        // 開催日時
  category: string,           // カテゴリ（ゲーム、音楽、アニメ、ライブ、推し活）
  creatorId: string,          // 作成者ID
  participantsCount: number,   // 参加者数（集約済み）
  likesCount: number,         // いいね数（集約済み）
  commentsCount: number       // コメント数（集約済み）
}
```

### commentsコレクション（コメントデータ）
```
{
  countdownId: string,        // 対象カウントダウンID
  content: string,           // コメント内容
  authorId: string,          // 投稿者ID
  authorName: string,        // 投稿者名
  createdAt: timestamp,      // 投稿日時
  likesCount: number,        // コメントいいね数
  repliesCount: number       // 返信数
}
```

### countdownLikesコレクション（いいね管理）
```
{
  countdownId: string,       // 対象カウントダウンID
  userId: string,           // いいねしたユーザーID
  createdAt: timestamp      // いいね日時
}
```

### trendRankingsコレクション（事前計算済みランキング）
```
{
  countdownId: string,      // 対象カウントダウンID
  eventName: string,        // イベント名
  category: string,         // カテゴリ
  eventDate: timestamp,     // 開催日時
  participantsCount: number, // 参加者数
  commentsCount: number,    // コメント数
  likesCount: number,       // いいね数
  trendScore: number,       // 算出されたトレンドスコア
  rank: number,            // ランク
  updatedAt: timestamp     // 更新日時
}
```

## ⚡ クエリ最適化の実装状況

### ✅ 実装済み最適化
1. **集約フィールドの使用**
   - `countsコレクション`に`participantsCount`, `likesCount`, `commentsCount`を集約保存
   - 表示時の追加クエリを不要に

2. **Cloud Functions による自動集約**
   - コメント投稿時: `onCommentCreate`でcommentsCountを自動更新
   - いいね時: `onLikeCreate`でlikesCountを自動更新
   - クライアント側での集計計算を廃止

3. **事前計算済みランキング**
   - `trendRankings`コレクションに計算済みデータを保存
   - 5分間隔でCloud Functionsが自動更新
   - ランキング表示時は事前計算データを高速取得

4. **最適化されたStreamBuilder**
   - `OptimizedStreamService`でキャッシュ機能を実装
   - デバウンス処理で不要な更新を削減
   - バッチ化で連続更新を効率化

5. **分散カウンターパターン**
   - 高負荷時のホットスポット問題を回避
   - `countdownShards`コレクションで書き込み競合を分散

### 📊 パフォーマンス改善効果

#### クエリ数削減
- **従来**: カウントダウン表示時にコメント数・いいね数を個別取得（N+1問題）
- **現在**: 集約済みデータを1回のクエリで取得
- **削減率**: 約90%のクエリ数削減

#### リアルタイム処理の効率化
- **トレンドランキング**: 定期バッチ更新（5分間隔）で負荷分散
- **カウント更新**: Cloud Functionsによるサーバーサイド処理
- **キャッシュ活用**: 同一データの重複読み込み防止

## 🔄 データフローの仕組み

### いいね機能
1. ユーザーがいいねボタンをタップ
2. `countdownLikes`コレクションにレコード作成
3. Cloud Functions `onLikeCreate`が自動実行
4. `counts`ドキュメントの`likesCount`が自動更新
5. リアルタイムでUIに反映

### コメント投稿
1. ユーザーがコメントを投稿
2. `comments`コレクションに新レコード作成
3. Cloud Functions `onCommentCreate`が自動実行
4. `counts`ドキュメントの`commentsCount`が自動更新
5. リアルタイムでコメント一覧とカウントが更新

### ランキング更新
1. Cloud Schedulerが5分ごとに`updateTrendRankings`を実行
2. 全`counts`ドキュメントから集約値を取得
3. トレンドスコア計算: `参加者数×1.0 + コメント数×2.0 + いいね数×1.5`
4. 開催日程による重み付け（直前のイベントほど高スコア）
5. `trendRankings`コレクションを一括更新
6. アプリ側では事前計算済みデータを高速表示

## 🔒 セキュリティ設計

### Firestore Security Rules
- **読み取り**: 基本的に全ユーザーが可能
- **書き込み**: 認証ユーザーのみ
- **削除**: 作成者本人のみ
- **管理データ**: Cloud Functionsからのみ書き込み可能

### 認証方式
- **現在**: 匿名認証（開発段階）
- **本格運用時**: Google・Apple・メールアドレス認証への移行予定
